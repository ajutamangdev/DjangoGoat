Security Reference Guide

Essential security practices organized by vulnerability type and implementation area

XSS Prevention

Cross-Site Scripting

Never use |safe with user input
Use Django forms for input validation
Implement Content Security Policy (CSP)
Use |escape for explicit escaping
Validate and sanitize all user inputs

File Upload Security

Secure File Handling

Validate file extensions AND content (magic bytes)
Store files outside web root
Generate random filenames (UUID)
Set file size limits
Scan uploaded files for malware

Authentication

User Access Control

Use Django's built-in authentication
Use @login_required decorator
Configure strong password validators
Implement rate limiting for login attempts
Use two-factor authentication (2FA)

Database Security

Data Protection

Always use Django ORM (prevents SQL injection)
Use parameterized queries for raw SQL
Apply least-privilege database permissions
Encrypt sensitive database fields
Regular database backups

HTTPS & Transport

Secure Communication

Force HTTPS in production
Enable HSTS headers
Use secure cookie flags
Implement certificate pinning
Use TLS 1.2+ only

Sessions & Cookies

Session Management

Set SESSION_COOKIE_SECURE = True
Set SESSION_COOKIE_HTTPONLY = True
Configure session timeout
Use SameSite cookie attribute
Regenerate session ID on login
